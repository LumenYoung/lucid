# Lucid Maintenance

This document covers the operator-side maintenance workflows for a deployed Lucid service.

These scripts live under `deploy/maintenance/`, but the usage guidance lives here so the repo's
human-facing documentation stays centralized under `docs/`.

## Scope

The maintenance scripts are for operators with direct access to the Lucid runtime, Graphiti
configuration, and FalkorDB data plane. They are not part of the normal agent integration path.

Files:

- `deploy/maintenance/reembed_group.py`
- `deploy/maintenance/replay_group.py`
- `deploy/maintenance/common.py`

## Required Inputs

Use the same config and env material as the Lucid runtime:

- `--config /path/to/config-docker-falkordb.yaml`
- `--env-file /path/to/lucid-graphiti.env`
- `--graphiti-mcp-server-path /path/to/graphiti/mcp_server`

If `GRAPHITI_MCP_SERVER_PATH` is already set, you can omit `--graphiti-mcp-server-path`.

If you run the scripts from the host shell instead of from inside the compose network, the env file
may contain a container-only FalkorDB hostname. In that case, export a host-routable override first:

```bash
export FALKORDB_URI=redis://<reachable-falkordb-address>:6379
```

Shell overrides intentionally take precedence over the env file.

## Re-embed In Place

Use this when only the embedding model changed and you want to keep the current graph structure.

```bash
uv run --project /path/to/graphiti/mcp_server \
  python deploy/maintenance/reembed_group.py \
  --config deploy/graphiti/config-docker-falkordb.yaml \
  --env-file deploy/env/lucid-graphiti.env \
  --graphiti-mcp-server-path /path/to/graphiti/mcp_server \
  --group-id main
```

Use `--dry-run` first if you only want the affected counts.

## Export And Replay

Use this when you want a destructive rebuild from stored episodes, or when the extraction behavior
changed enough that the graph should be regenerated from episodic source material.

Export:

```bash
uv run --project /path/to/graphiti/mcp_server \
  python deploy/maintenance/replay_group.py export \
  --config deploy/graphiti/config-docker-falkordb.yaml \
  --env-file deploy/env/lucid-graphiti.env \
  --graphiti-mcp-server-path /path/to/graphiti/mcp_server \
  --group-id main \
  --snapshot-file /tmp/lucid-main.snapshot.json
```

Replay into another group:

```bash
uv run --project /path/to/graphiti/mcp_server \
  python deploy/maintenance/replay_group.py replay \
  --config deploy/graphiti/config-docker-falkordb.yaml \
  --env-file deploy/env/lucid-graphiti.env \
  --graphiti-mcp-server-path /path/to/graphiti/mcp_server \
  --snapshot-file /tmp/lucid-main.snapshot.json \
  --target-group-id main-rebuild \
  --clear-target
```

Single-command roundtrip:

```bash
uv run --project /path/to/graphiti/mcp_server \
  python deploy/maintenance/replay_group.py roundtrip \
  --config deploy/graphiti/config-docker-falkordb.yaml \
  --env-file deploy/env/lucid-graphiti.env \
  --graphiti-mcp-server-path /path/to/graphiti/mcp_server \
  --source-group-id main \
  --target-group-id main-rebuild \
  --snapshot-file /tmp/lucid-main.snapshot.json \
  --clear-target
```

## Limitations

- `replay_group.py` rebuilds from episodes. It does not preserve community nodes or fact edges
  directly; those are regenerated by the current Graphiti pipeline.
- `replay_group.py` regenerates episode UUIDs on rebuild. Graphiti's `add_episode()` path cannot
  seed a missing episode UUID into an empty target graph.
- Saga-specific linkage is not exported today. Treat replay as a graph rebuild from episodic
  content, not a byte-for-byte restore.
- `reembed_group.py` still needs a working embedder provider because it writes fresh vectors
  directly.
